  ---
  Now regarding your question about optimizing AUR build times:

  Yes, there are two main things you can do:

  1. MAKEFLAGS in /etc/makepkg.conf — sets parallel compilation jobs:
  MAKEFLAGS="-j$(nproc)"
  The rule of thumb is -j$(nproc) which uses all available CPU cores. Some
  people use nproc + 1 or nproc + 2.

  2. BUILDDIR in /etc/makepkg.conf — compile in tmpfs (RAM) instead of disk:
  BUILDDIR=/tmp/makepkg
  This avoids disk I/O bottleneck during compilation.

  3. Compiler optimizations in /etc/makepkg.conf:
  - CFLAGS / CXXFLAGS: can add -march=native to optimize for your specific CPU
  - RUSTFLAGS: -C target-cpu=native for Rust packages
  - COMPRESSZST threads: use zstd -T0 to use all cores for package compression

  4. ccache — caches compiled objects so rebuilds are faster:
  BUILDENV=(... ccache ...)

  Want me to create a makepkg-optimization role with all of these? The
  -j$(nproc) and tmpfs build dir are the biggest wins. I can show you the full
  task list before implementing.

⏺ Not quite — those are needed during the install:

  - grub-install writes the GRUB bootloader to the EFI partition. Without it,
  the system won't boot at all after reboot. Can't run this from Ansible later
  because you'd never get to a booted system.
  - grub-mkconfig generates the initial boot config. Without it, grub-install
  alone won't know which kernel to boot.
  - mkinitcpio -P builds the initial ramdisk. Without it, the kernel can't mount
   the root filesystem at boot. The HOOKS line tells it how to find the disk
  (keyboard, block, filesystems).
